Тестирование контрактов системы DaData/Яндекс.Геокодер/GigaChat
=========================================

### Задача

Ниже приведен сравнительный анализ систем Яндекс.Геокодер, DaData, GigaChat для использования для прямого и обратного геокодирования. В качестве данных использован файл 

### Анализ тестирования 

Для проведения тестирования был написан скрипт, который выполнял запрос к системам и получал результаты. Запросы выполнялись как по прямому, так и по обратному геокодированию. Всего было выполнено по 100 запросов на прямо и обратное геокодирование в каждом сервисе

По прямому геокодированию были получены следующие результаты:

#### Яндекс.Геокодер
##### Прямое геокодирование
Среднее время обработки запроса с Яндекс.Геокодера по прямому геокодированию составило 0.2 секунды.

Для оценки качества результатов использовалось поле precision, которое возвращает Яндекс Геокодер.

В данных, которые возвращает Яндекс.Геокодер, присутствуют следующие понятия точности:

exact - найден дом с указанным номером дома.
number - найден дом с указанным номером, но с другим номером строения или корпуса.
near - найден дом с номером, близким к запрошенному.
range - найдены приблизительные координаты запрашиваемого дома.
street - найдена только улица.
other - не найдена улица, но найден, например, посёлок, район и т. п.

В прямом кодировании совпадения координат составили:
exact - 84 раз.
number - 14 раз.
near -  2 раза.

Точность прямого кодирования с помощью сервиса Яндекс.Геокодер является высокой (84%).

##### Обратное геокодирование
Среднее время обработки запроса с Яндекс.Геокодера по прямому геокодированию составило 0.20 секунд.

Аналогично с прямым геокодированием, совпадения адресов составили:
exact - 20 раз.
street - 43 раза.
other - 37 раз.

Точность обратного геокодирования с Яндекс.Геокодером не является высокой.

#### GigaChat
##### Прямое геокодирование
Среднее время обработки запроса с GigaChat по прямому геокодированию составило 0.24 секунд.

GigaChat самый сложный из сервисов, который участвовал в анализе. Ответы GigaChat являются приближенными, но лишенными точности. В некоторых случаях GigaChat ошибается и выдает совершенно не те результаты, либо же определяют только улицу. Понятия точности сопоставлены с понятиями у Яндекс.Геокодер. Совпадения адресов составили:
exact - 14 раз.
street - 60 раз.
other - 26 раз.

Точность прямого кодирования с помощью сервиса GigaChat является низкой.

##### Обратное геокодирование
Среднее время обработки запроса с GigaChat по прямому геокодированию составило 0.41 секунд.

Аналогично с прямым геокодированием, совпадения адресов составили:
exact - 3 раза.
street - 62 раза.
other - 35 раз.

Точность обратного геокодирования с помощью сервиса GigaChat является низкой.

#### DaData
##### Прямое геокодирование
Среднее время обработки запроса с DaData по прямому геокодированию составило 0.18 секунд.

Для оценки качества прямого геокодирования использовалось поле qc_geo, которое возвращает DaData.

DaData имеет свою шкалу оценивания точности геокодирования. Она разделяется на следующие категории точности:
0:  "Точные координаты дома",
1:  "Ближайший дом",
2:  "Улица",
3:  "Населенный пункт",
4:  "Город",
5:  "Координаты не определены"

Точные координаты дома - 83 раза.
Ближайший дом - 13 раз.
Улица - 4 раза.
Данные результаты показывают, что в случаях, когда сервис находил нужный адрес, он возвращал координаты высокой точности.
Точность прямого кодирования с помощью сервиса DaData является очень высокой.

##### Обратное геокодирование
Среднее время обработки запроса с DaData по прямому геокодированию составило 0.12 секунд.

По сравнению с прямым геокодированием обратное выполнялось быстрее.

Для оценки качества обратного геокодирования использовалось поле qc_geo.

Аналогично с прямым геокодированием, совпадения адресов составили:
Точные координаты дома - 70 раза.
Ближайший дом - 1 раз.
Улица - 7 раз.
Населенный пункт - 4 раза.
Город - 14 раз.
Координаты не определены - 4 раза, они не участвовали в статистике. Для 4 запросов сервис вернул в ответе quality.

Точность обратного геокодирования с помощью сервиса DaData является высокой.

### Результаты тестирования

- DaData рекомендуется использовать в системах, где ключевыми требованиями являются высокая скорость обработки запросов и получение подробной структурированной информации об адресе.
- Яндекс Геокодер предпочтителен в задачах, где важны максимальная полнота геокодирования, устойчивость к различным вариантам записи адресов и высокая вероятность успешной обработки сложных или неоднозначных запросов.
- GigaChat целесообразно применять только как вспомогательный инструмент для обработки текстов на естественном языке. Использовать его в качестве самостоятельного сервиса геокодирования не рекомендуется, поскольку модель не предназначена для определения координат и не всегда соблюдает требуемый формат ответа.

### Критерии приемки

#### Процесс и контекст использования
Документ применяется при выборе сервиса геокодирования. Используется после проведения тестирования для анализа полученных результатов и обоснования технического решения.

#### Цель создания
Провести сравнительный анализ DaData, Яндекс Геокодера и GigaChat по установленным критериям и определить наиболее подходящий сервис.

#### Что становится определено
Документирует результаты тестирования, включая среднее время отклика, точность геокодирования, уровень детализации данных, особенности работы каждого сервиса и итоговое решение по выбору.

#### Пользователи артефакта
Системный аналитик, архитектор, разработчики и технический руководитель. Документ используется для обоснования выбора сервиса и принятия архитектурных решений.

#### Использование в дальнейшем
Служит основой при проектировании интеграции, формировании требований и реализации взаимодействия с выбранным сервисом геокодирования.

#### Последствия отсутствия
Отсутствие документа может привести к выбору сервиса без объективного сравнения, что повышает вероятность принятия неэффективного решения и увеличивает риск последующих изменений в системе.
